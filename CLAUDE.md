# ワタシのトリセツ 開発ルール

## プロダクトの核

友達の回答で、自分のトリセツが完成していく大学生向けの自己理解サービス。

## ターゲット

軽い自己理解ニーズのある大学生。
就活ガチ勢ではなく、友達関係・恋愛・SNSネタ・キャラ診断寄り。

## MVPで作るもの

- LP
- 自己評価15問
- 仮トリセツ表示
- 友達招待URL生成
- LINE共有ボタン
- 友達5問回答
- 1人回答で部分表示
- 3人回答で完成版トリセツ
- 5人回答で深掘りレポート
- SNSシェア画像
- LINE登録導線
- KPI計測

## MVPで作らないもの

- AIチャット
- Premium課金
- BtoB組織診断
- 高校生版・社会人版の本格実装
- 10人/20人アンロック
- 複雑なLINE通知
- Compass色の強い送客機能

## 技術スタック

- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- Vercel
- GitHub

## UIトーン

- モバイルファースト
- ポップな取扱説明書風
- 温かいオフホワイト + タイプ別カラー
- ラベル、スタンプ、注意書き、チェックリスト、ゆるい警告アイコン
- ネガティブ表現は「愛されるクセ」に変換する
- 友達に送っても恥ずかしくない軽さを優先する

## 最重要判断基準

診断後に、友達3人へ送りたくなるか。

## 実装ルール

- まず既存コードを確認してから変更する
- 仕様を勝手に広げない
- 1回の変更は1機能単位にする
- モバイルファーストで実装する
- エラー表示、ローディング表示、空状態を用意する
- TypeScriptの型を崩さない
- 自由記述や友達回答は、本人を傷つける形で表示しない
- MVP外の機能は、提案しても実装しない

## KPIイベント

最低限、以下を計測できるようにする。

- diagnosis_started
- diagnosis_completed
- friend_invite_clicked
- friend_answer_completed
- three_friends_unlocked
- line_registered
- share_clicked

## エージェント追加ルール

Claude Codeは、必要だと判断した場合のみ .claude/agents/ にプロジェクト専用エージェントを追加してよい。

ただし、最初から大量に作成しないこと。
MVP初期では原則1〜3体までに抑えること。

新しいエージェントを作る場合は、以下を明記すること。

- 役割
- 使用タイミング
- 作成理由
- CLAUDE.mdとの違い
