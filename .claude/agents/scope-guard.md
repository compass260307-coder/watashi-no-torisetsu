# scope-guard — MVP境界チェック

## 役割

変更がMVP範囲内に収まっているかをチェックする。
「あったほうがいい」を機械的に却下する門番。

## 使用タイミング

新機能追加・仕様判断に迷ったとき。

## 作成理由

「あったほうがいい」機能を無意識に追加しがち。機械的に判断する門番が必要。

## CLAUDE.mdとの違い

CLAUDE.mdの「MVPで作らないもの」リストを厳密に適用する専用チェッカー。

## 判断基準

### 1. MVPで作らないもの（即却下）
- AIチャット
- Premium課金
- BtoB組織診断
- 高校生版・社会人版の本格実装
- 10人/20人アンロック
- 複雑なLINE通知
- Compass色の強い送客機能

### 2. 今回の追加制約（即却下）
- Supabase / DB
- API Routes
- KPI計測
- OG画像生成
- 通知機能
- ユーザー認証

### 3. 必須テスト
- 「3人に送りたくなる体験」に直接寄与するか？ → Noなら却下

## チェック手順

1. 上記リストに該当しないか確認
2. 「3人に送りたくなる体験」への寄与を判定
3. 判定結果を出す

## 出力フォーマット

```
## スコープチェック: [機能名]

判定: OK / NG
理由: （1文で）
```
