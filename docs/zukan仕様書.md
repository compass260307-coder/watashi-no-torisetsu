# /zukan-internal 図鑑ページ 実装仕様書

CCへ。本番の `/zukan-internal`(src/app/zukan-internal/page.tsx)を、
社内デバッグ表から「32キャラ図鑑」に作り込む。下記の確定仕様で実装してほしい。

---

## 0. 前提・スコープ
- **軽量版**で作る。本文(恋愛/取説/付き合い方)はカードに載せない。索引に徹し、
  詳細を見たい場合は各カードから `/preview/[id]`(フル結果ページ)へ飛ばす。
  → love-by-type-32 / self-result-32 / manual-by-type-32 の繋ぎ込みは**不要**。
- データソースは現状どおり character-32.ts 系のリゾルバのみ(thirty-two-types.ts経由)。
- このページは Notion DB とは非連携のまま(今回スコープ外)。

---

## 1. ★重要な前提の訂正(旧実装と変わった点)
- **動物名(「れいせいシャチ」等の"形容詞＋動物")は廃止済み。今後一切使わない。**
- **essence(先導者・探偵・将軍…)＝キャラ名。これが唯一の名前で、カードの主役。**
  旧実装の「称号」は、もう"称号"ではなく"キャラ名"として扱う。
- よって旧 character-32.ts の name フィールド(動物名)はカードに出さない。
  essence をキャラ名として表示する。

---

## 2. カード1枚の構成(確定・この順番で縦に)
各キャラカードに、上から順に:

1. **画像**(public/characters/v3/<slug>.png。thirtyTwoImagePath(id)。サイズは図鑑として
   見やすく現状72pxより大きめ推奨=128px前後。実装側で調整可)
2. **キャラ名**(＝essence。先導者・探偵 等。カードの主役。一番大きい文字)
3. **Big Fiveコード**(下記§3の大文字/小文字方式)
4. **MBTIコード**(ENFJ_R 等。MBTI4文字＋_R/_N)
5. **一言説明文**(★文言は後日決定。今はプレースホルダでよい。
   暫定で oneLiner を流用するか、空欄＋「(説明文は後日)」等の仮置きでOK。
   実装上は character-32 に新フィールドを足すか、暫定で oneLiner を表示する形を提案してほしい)
6. **結果ページURL**(`/preview/<id>` へのリンク。例 /preview/sparkle-dolphin__R。
   クリックでそのキャラのフル結果ページへ)
7. **画像パス**(`/characters/v3/<slug>.png` をテキストで表示。SNS/HP用に手元で
   パスをコピーできるように。コピー可能だと尚良い)

旧カードにあった oneLiner / 内部32ID / 群内位置(C× × A×)の扱い:
- 内部32ID は §6 のURL・画像パスで実質カバーされるので、単独表示は任意(残してもよい)。
- oneLiner は §5 の一言説明文が確定するまで暫定流用してよいが、確定後は説明文に置換。
- 群内位置(C× A×)は図鑑では不要(削ってよい)。

---

## 3. ★Big Fiveコードの表示方式(大文字/小文字)
- 5軸 **O C E A N** の順で5文字を並べる。
- **その軸が高ければ大文字、低ければ小文字。**
- 各軸の高低の出どころ:
  - O(開放): MBTIの N/S。N→高(O大文字) / S→低(o小文字)
  - C(誠実): MBTIの J/P。J→高(C) / P→低(c)
  - E(外向): MBTIの E/I。E→高(E) / I→低(e)
  - A(協調): MBTIの F/T。F→高(A) / T→低(a)
  - N(神経症): 接尾辞。_N→高(N) / _R→低(n)
- 例:
  - 先導者 ENFJ_R(N S→O高, J→C高, E→E高, F→A高, _R→N低)= **OCEAn**
  - 全軸高い繊細タイプ = OCEAN
- ⚠ 注意: この "N"(末尾)は Big Five の神経症。MBTIコード(ENFJ_R)の "N"(直感)とは
  別物。カード上で MBTI と Big Fiveコードが並ぶと N が2回出るが、これは仕様として正しい。
  既存の符号表記(O+ C+ E+ A+ N−)があれば、この大文字小文字方式に置き換える。

---

## 4. ★グループ色バグの修正(本実装に同梱)
現状 GROUPS 配列にハードコードされた見出しバー色が2件間違っている。修正必須。
- 空 sky: ハードコード #A8D88A(緑) → 正 #F3DF7A(黄)
- 陸 land: ハードコード #F5A9C0(ピンク) → 正 #A8D88A(緑)
- 海 sea / 未知 unknown は一致済み。
**修正方針: GROUPS のハードコード色を撤去し、正典 THIRTY_TWO_GROUP_COLOR 参照に一本化。**
ヘッダ説明文の「🕊 空 (緑)」「🌿 陸 (ピンク)」等の誤記も正しい色名に直す。

---

## 5. ★色の使い分け(2系統を併用)
色は2系統ある。図鑑では以下のように使い分ける:
- **グループ見出しバー** = アクセント色(THIRTY_TWO_GROUP_COLOR。空=黄/陸=緑/海=青/未知=紫)。はっきり示す。
- **各カードの背景** = 背景ウォッシュ色(淡い。preview/all の GROUP_BG と同じ:
  空=#FDEFB4 / 海=#BEF2F9 / 陸=#D8F2C0 / 未知=#E7DCFB)。画像が映える淡色。
- **N軸バッジ**(繊細/鉄壁) = アクセント色(現状どおり thirtyTwoColor(id))。
※ ウォッシュ色は preview/all から定数を共有するか、図鑑側で同値を定義してよい(実装判断に委ねる)。

## 6. レイアウト
- 現状の構造(4グループ縦積み、各グループ内で base16=C×A の4ペアを __N/__R 並べる)は
  踏襲してよい。グループ順も現状(空→陸→海→未知)or 任意で可。
- フィルタ・並べ替え・検索は今回不要(静的全件列挙でよい)。
- noindex の隠しルートのまま(導線なし)で継続。

---

## 7. 今回やらないこと(スコープ外・申し送り)
- **catchphrase の性格ズレ修正**(フェススター ENFP_R 等4件)は別タスク。図鑑実装には含めない。
  ※catchphrase はそもそも軽量版カードに出さないので、図鑑表示には影響しない。
- **一言説明文(§5)の文言決め**は後日(言葉を練る別作業)。今はプレースホルダ実装まで。
- **Notion DB との同期**は今回スコープ外。
- ENFJ_R の catchphrase「その熱は、人の心に火をともす」への変更は別途反映済み/要確認だが、
  catchphrase は図鑑に出さないため本実装には無関係。

## 8. 実装後の確認依頼
- ローカルで /zukan-internal を開き、報告してほしい:
  1. 各カードに [画像・キャラ名(essence)・Big Fiveコード(OCEAn形式)・MBTI・一言説明文(暫定)
     ・/preview リンク・画像パス] が出ているか。
  2. グループ見出しの色が正しい(空=黄・陸=緑・海=青・未知=紫)か。
  3. カード背景がウォッシュ色になっているか。
  4. 動物名(れいせいシャチ等)がどこにも出ていないか。
  5. Big Fiveコードが大文字小文字で正しいか(先導者=OCEAn 等、代表数体で検算)。
- tsc/eslint クリアを確認。OKなら commit/push。

# 追記2（§5・了承済みの申し送り）

- 一言説明文は暫定で oneLiner を流用してよいが、海・陸グループの oneLiner は
  旧マスコット（動物）基準で書かれており新画像と内容がズレている点は承知済み。
  暫定表示でOK（後日の文言確定タスクで差し替える前提）。プレースホルダである旨が
  分かる表示（淡色/「(説明文は後日)」併記等）だと尚良い。
