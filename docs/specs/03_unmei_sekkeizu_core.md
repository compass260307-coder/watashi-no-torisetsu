# 実装指示書③:「運命の設計図」コンテンツ本体

## 背景

ワタシのトリセツの新有料商品。西洋占星術のネイタルチャート(出生図)を実計算し、
Big Five診断結果・32タイプ称号と掛け合わせたAI鑑定文+出生図ホイール+簡易経過チャート(トランジット)を提供する。

ポジショニング:「診断はガチ、占いは遊び。ただし遊びの中身は性格データで書いてある」。
必ずエンタメである旨を明示し、科学的診断本体の信頼性と混同させない。

## 商品・価格構成

| 商品 | 価格 | 内容 |
|---|---|---|
| 穴抜き解放(既存) | ¥499 | 現行のまま変更なし |
| 運命の設計図 | ¥1,980 | 穴抜き解放の全内容 + 出生図ホイール + AI鑑定 + 簡易経過チャート |
| アップグレード | ¥1,480 | ¥499購入済みユーザー向け差額。内容は¥1,980と同一 |

- ¥1,980 は ¥499 の内容を**包含**する(全部入り)。購入判定は `entitlements` で管理:
  `unmei` 権限を持つユーザーは `499` 権限も自動的に満たす
- 決済:既存の Stripe(カード)+ PayPay フローを流用。新規に2つの Price を作成
  (¥1,980 / ¥1,480)。¥1,480 は¥499購入済みユーザーにのみ購入導線を表示し、
  サーバー側でも購入資格を検証する(URLを知っていても未購入者は¥1,480を買えない)
- 特商法ページ・領収書まわりは既存の仕組みに新商品を追記

## アーキテクチャ概要

```
[出生データ入力(指示書②)]
        ↓
[① エフェメリス計算]  … サーバー内・決定論的・無料
        ↓ chart JSON
[② チャート保存]      … natal_charts テーブル
        ↓ (購入時)
[③ AI鑑定生成]        … Claude API。チャートJSON + Big Five + タイプ称号を入力
        ↓
[④ キャッシュ保存]    … natal_readings(不変・永久キャッシュ)
        ↓
[⑤ 表示]              … 出生図ホイールSVG + 鑑定文 + 経過チャート
```

**原則:計算はコードが行い、AIは解釈だけを書く**(天体位置をAIに推測させない)。

## ① エフェメリス計算

- ライブラリ:`circular-natal-horoscope-js`(第一候補)または swisseph 系。
  選定はライブラリの精度・メンテ状況を確認して決定し、READMEに選定理由を記録
- 入力:生年月日、出生時刻(不明なら12:00)、緯度経度、タイムゾーン(日本出生前提でJST固定。
  将来の海外対応はスコープ外)
- 出力(chart JSON):
  - 太陽・月・水星・金星・火星・木星・土星の星座とサイン内度数
  - ASC(上昇宮)・MC の星座と度数 ※時刻不明時はASC/MCを出力せず `houses_available: false`
  - 主要アスペクト(合0°・衝180°・トライン120°・スクエア90°・セクスタイル60°、オーブは合8°/他6°程度)
- 検証用フィクスチャ:2004-04-26 02:00 JST・北九州(33.883N, 130.883E)で
  太陽=牡牛座5.8°、月=蟹座13.4°、ASC=水瓶座16.4°、MC=射手座2.0° 前後になることをテストで確認

## ② ③ ④ データモデルと生成

```sql
create table natal_charts (
  user_id uuid primary key references auth.users(id) on delete cascade,
  chart jsonb not null,          -- 上記chart JSON
  houses_available boolean not null,
  computed_at timestamptz not null default now()
);

create table natal_readings (
  user_id uuid primary key references auth.users(id) on delete cascade,
  reading jsonb not null,        -- 生成済み鑑定(セクション構造で保存)
  model text not null,
  generated_at timestamptz not null default now()
);

create table transit_readings (
  user_id uuid not null references auth.users(id) on delete cascade,
  month text not null,           -- '2026-07' 形式
  reading jsonb not null,
  generated_at timestamptz not null default now(),
  primary key (user_id, month)
);
```

- RLS:本人select、書き込みはサービスロールのみ
- **キャッシュ規律(重要)**:
  - 出生図鑑定は不変 → `natal_readings` があれば**絶対に再生成しない**
  - 経過チャートは (user, month) 単位で1回生成 → 同月内は再利用
  - 生成はサーバーサイドAPIルートで実行。クライアントから直接Anthropic APIを叩かない
- Claude API:モデルは環境変数で切替可能に(初期値 claude-sonnet-4-6)。
  出力はJSONで受け取り(プロンプトで「JSONのみ返す」を厳格指定)、parse失敗時は1回リトライ
- 生成タイミング:購入完了 webhook 受信時に生成開始。表示画面は生成中ローディング
  (「あなたの空を読んでいます…」+ホイール回転アニメ)を出し、ポーリングまたはリアルタイム購読で完了を待つ

## AI鑑定のプロンプト設計(プロンプトはコード内に定数化し、別ファイルに分離)

**システムプロンプトに含める要素:**

1. 話者設定:ユニコーン(未知のなかま・星を読む係)とタカ・ストラテジスト(戦略を立てる係)の
   二人の案内人が語る体。基本はユニコーンが星を語り、「どう活かすか」の段でタカが引き取る
2. トーンガイド(grace文体):
   - 欠点を直させない。「すでにあるものを受け取る」語り
   - 断定的な運命論を避ける:「星が示すのは運命ではなく素質。どう使うかを決めてきたのがあなた」
   - 診断(Big Five)と星の関係は2分岐で書く:
     - **呼応**:一致している場合 →「星と診断が、別々の道から同じことを言っている」
     - **選択**:ズレている場合 →「星の素質を、あなたはこう生きることを選んだ」
   - どちらに転んでも物語として成立させる(この分岐指示を明示的にプロンプトへ入れる)
3. 禁止事項:医療・健康・金銭の具体的助言、不安を煽る予言、死・病気・事故への言及、
   実在人物への言及。ネガティブな配置も必ず前向きな受け取り方に変換する
4. 入力データ:chart JSON、Big Five 5因子スコア、32タイプ称号、`houses_available`、
   時刻不明時は「ASCに触れない」指示

**出力構造(natal reading):**

```json
{
  "hitokoto": "無料ティーザー用の1〜2文",
  "sections": [
    { "id": "haichi",  "title": "あなたという星の配置", "body": "..." },
    { "id": "kokoro",  "title": "心の天気",             "body": "..." },
    { "id": "chosen",  "title": "挑戦の風向き",         "body": "..." },
    { "id": "grace",   "title": "最後にひとつだけ",     "body": "..." }
  ]
}
```

- 全体で800〜1200字目安。「最後にひとつだけ」は既存の恋愛セクションと同じ締めの型を踏襲

**経過チャート(transit)の出力構造:**

```json
{
  "month": "2026-07",
  "headline": "今月の空のひとこと",
  "movements": [
    { "planet": "木星", "aspect": "あなたの太陽とトライン", "body": "解釈テキスト" }
  ],
  "lucky_action": "今月のラッキーアクション1つ"
}
```

- transit計算:当月1日時点の主要天体(木星・土星は必須、火星・金星は任意)と
  出生図天体とのアスペクトをコードで算出し、上位2〜3個だけAIに解釈させる

## ⑤ 表示画面

ルート:`/unmei`(仮。ボトムナビには今回追加しない)

- **未購入時**:ぼかしホイール+「今のひとこと」(hitokoto、出生データ入力済みの場合のみ生成せず
  定型ティーザー文でも可)+価格+購入CTA。インタースティシャル(指示書①)のCTA遷移先はここ
- **購入済み時**:
  1. 出生図ホイール(SVG):十二星座リング+天体グリフ配置+ASC/MC軸。
     生息地カラー(空#FDEFB4/陸#D8F2C0/海#BEF2F9/未知#E7DCFB)を火地風水の
     エレメント塗り分けに使用。時刻不明時はASC/MC軸非表示+注記
  2. 天体一覧(太陽・月など7天体の星座を簡潔に)
  3. AI鑑定文(sections を順に表示。話者アイコンとしてユニコーン/タカの小アイコンを併用可)
  4. 経過チャート:「{当月}の空」セクション。月が変わったら初回アクセス時に自動生成
  5. フッター注記:「本コンテンツはエンタメ目的です。医療・法律・投資等の助言ではありません」
- ホイールSVGはコンポーネント化(props: chart JSON)。描画ロジックにテストを付ける
  (代表チャートでの各天体の描画角度を検証)

## コスト・運用ガード

- 生成失敗時のリトライは最大2回、それでも失敗なら管理者通知+ユーザーには
  「時間をおいて再度お試しください」を表示(課金済みなので必ず後から取得できること)
- 月次で transit 生成数と API コストをログ集計(想定:1鑑定あたり10〜15円)
- レートリミット:同一ユーザーの生成APIは1分1回まで

## 計測イベント

- `unmei_lp_view` / `unmei_purchase_start` / `unmei_purchase_complete`(¥1,980 / ¥1,480 を区別)
- `unmei_reading_view` / `unmei_transit_view`
- ファネル:interstitial_cta → lp_view → purchase_start → complete を接続できるようにする

## 受け入れ条件

- [ ] フィクスチャの出生データで天体位置が期待値と一致する(自動テスト)
- [ ] ¥1,980購入で穴抜きコンテンツも解放される(entitlement包含)
- [ ] ¥1,480はサーバー側で¥499購入済みユーザーのみ決済可能
- [ ] 鑑定は購入後1回だけ生成され、以降はキャッシュ表示(APIが再呼び出しされないことをログで確認)
- [ ] 月替わりで経過チャートが自動更新される
- [ ] 時刻不明ユーザーでASC/MCなしの表示が破綻しない
- [ ] 出生データ削除(指示書②)で readings も連動削除される
- [ ] エンタメ注記が未購入・購入済み両画面に表示される
- [ ] Stripe / PayPay 両方でテスト決済がエンドツーエンドで通る
