# 実装指示書①:占いインタースティシャル(全画面差し込み)

## 背景

ワタシのトリセツに新商品「運命の設計図」(ホロスコープ占い × 性格診断、¥1,980)を追加する。
既存の¥499ペイウォール(穴抜き解放)のCVR約2%を守るため、¥499セクションには一切手を入れず、
その**手前**に全画面インタースティシャルを1回だけ表示する。

狙い:¥1,980を先に見せる → ×で閉じる → 直後に現れる¥499が相対的に安く見える
(アンカリング / ドア・イン・ザ・フェイス)。¥1,980自体の直販も狙う。

## 表示トリガー

- 対象ページ:診断結果ページ(/me 系の結果表示ページ)
- 発火条件:**ページ全体の50%地点までスクロール到達** かつ **スクロールが停止して300ms経過**
  - スクロール中に被せない(誤タップ・苛立ち防止)
  - 50%地点は¥499ペイウォールセクションより上であることをコード上で保証する
    (ペイウォール要素の offsetTop を取得し、min(全体50%, ペイウォール手前) で判定)
- 頻度制御:**1ユーザー1回のみ**
  - localStorage キー `uranai_interstitial_shown` (値: ISO日時) で管理
  - ログインユーザーは Supabase 側にもフラグを保存し、端末をまたいでも再表示しない
  - ×で閉じた後の再訪では表示しない。受け皿は別途の常設セクション(将来実装、本指示書のスコープ外)
- 除外条件:
  - すでに¥1,980商品(またはアップグレード)購入済みのユーザー
  - ¥499の決済フロー進行中(Stripe/PayPayリダイレクト直後の復帰時など)

## 画面構成(モックv5準拠)

全画面オーバーレイ。背景は夜空の完全不透過(下のページは透けない)。
縦に4層のレイヤー構造(3人ゲートのパララックス実装パターンを流用可):

1. **星空層(最背面)**:紺のグラデ2トーン(#0D1330 上部 / #131A3A 下部)+ 星の点描。
   星は10〜14個、うち4〜5個が opacity 0.3⇔1.0 の明滅(周期2.2s / 3.0s の2系統でずらす)
2. **ホイール層**:画面上部中央やや右に「?」入りのぼかし出生図ホイール
   - 二重円(実線+破線)+ 十字/斜めの目盛り + アクセント点3つ(金1・薄紫2)
   - **40秒/1周でゆっくり回転**(transform: rotate、transform-origin は円の中心)
   - 中央テキスト:「? ? ?」+「あなたの出生図」
3. **キャラ層**:
   - **ユニコーン(左上・奥)**:とんがり帽子+星マントの占い師姿。浮遊アニメ 2.8s周期、
     translateY 0→-10px、rotate +4°⇔-2°。周囲に「星のカード」2枚が独立して漂う
     (3.8s / 4.4s 周期、translateY -7px + rotate 揺れ)
   - **タカ(右下・手前)**:望遠鏡を空に構えるストラテジスト。浮遊アニメ 3.4s周期(0.3s遅延)、
     translateY 0→-16px、rotate +2°⇔-3°。ユニコーンより1.2〜1.3倍大きく表示(遠近差)
   - 2体の視線が対角線上で交差する配置(天のユニコーン ↔ 見上げるタカ)
4. **UI層(最前面)**:
   - バッジ:「ホロスコープ占い × 性格診断」(薄紫 #E7DCFB 系ピル、opacity 0.75⇔1.0 の明滅 3.2s)
   - 見出し:「{タイプ称号}のあなたが\n生まれた瞬間の空」
     - {タイプ称号} はユーザーの32タイプ称号を動的差し込み(例:風雲児)。
       未診断・未ログイン時は「あなたが生まれた瞬間の空」にフォールバック
   - 価格行:「運命の設計図 ¥1,980」
   - CTAボタン:「運命の設計図を占う」(金 #F2B33D 系、角丸ピル)
     - **表示から3秒後に**鼓動アニメ開始(2.6s周期で scale 1→1.04→1 を1拍)。表示直後は静止
     - タップで占いLP/購入フローへ遷移(遷移先URLは環境変数 or 定数で差し替え可能に)
   - 注意書き(最下部・小):「エンタメコンテンツです・出生データは鑑定の計算のみに使用」

## ×ボタン(閉じる)

- 位置:右上 14px, 14px。直径34px以上、**タップ判定は44×44px以上**
- 表示と同時に出す(遅延表示・フェイク×は禁止)
- タップで即時クローズ(アニメーションは150ms以内のフェードアウトのみ)
- ×の周囲44pxの範囲では、キャラ(ユニコーン)のタップ判定を無効化する(誤タップ防止)
- 閉じたら頻度制御フラグを записし、下のページにそのまま戻る(スクロール位置維持)

## アニメーション実装要件

- すべて **transform / opacity のみ**で実装(GPU合成、reflow禁止)
- `@media (prefers-reduced-motion: reduce)` では全アニメーション停止(静止画として成立させる)
- 動きの階層を守る:
  - 大:キャラ浮遊・カード漂い
  - 緩:ホイール回転(40s)
  - 瞬:星・バッジの明滅
  - 稀:CTA鼓動(3秒後開始)
- 登場演出:オーバーレイ表示時、キャラ2体は translateY(+24px)+opacity 0 から
  400ms かけてふわっと出現(ユニコーン先行、タカ150ms遅延)

## アセット

以下を想定。ファイルが未支給の場合はプレースホルダ(単色シルエット)で実装し、差し替え可能な構造にする:

- `unicorn_base.png`:ユニコーン素体(透過、見上げ気味の角度)
- `unicorn_hat.png` / `unicorn_cape.png`:帽子・マント(別レイヤー、任意)
- `taka_base.png`:タカ素体(透過)
- `taka_wing.png`:タカの翼(別レイヤー、任意。存在する場合のみ羽ばたきアニメ:rotate ±14°、1.8s)
- `star_card_1.png` / `star_card_2.png`:星のカード(なければコードで描画したSVGカードで代替)
- 小道具レイヤーは素体の子要素として absolute 配置し、座標をコンフィグ化(後から調整しやすく)

## 計測イベント

以下を既存の計測基盤(GA4 / GTM)に送信:

- `uranai_interstitial_view`(表示)
- `uranai_interstitial_close`(×タップ)
- `uranai_interstitial_cta`(CTAタップ)
- 併せて、表示後の¥499購入イベントに `after_interstitial: true` フラグを付与
  (インタースティシャルが¥499のCVRに与える影響を検証するため)

## 受け入れ条件

- [ ] 診断結果ページで条件を満たすと1回だけ全画面表示される
- [ ] スクロール中には表示されない(停止300ms後に出る)
- [ ] ×で即時に閉じ、同一ユーザー(同一端末+ログイン時はアカウント単位)に再表示されない
- [ ] ¥1,980購入済みユーザーには表示されない
- [ ] タイプ称号が動的に差し込まれる/未診断時はフォールバック文言になる
- [ ] reduced-motion 環境で全アニメが停止し、静止状態でもレイアウトが崩れない
- [ ] Lighthouse でインタースティシャル表示によるCLSが発生しない(オーバーレイなのでレイアウトに影響しない)
- [ ] 計測イベント4種が発火する
- [ ] モバイル実機(iOS Safari / Android Chrome)で60fps相当の滑らかさ
