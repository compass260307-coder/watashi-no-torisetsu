## タスク1:出生データ未入力時のハンドリング(最優先・バグ修正)

現状、entitlement付与後に出生データが無いと鑑定生成が開始できず、
/unmei のローディングが永遠に終わらない。

- webhook の鑑定生成トリガーで、birth_profiles にレコードが無い場合は
  生成をスキップし、エラー扱いにしない(正常な待機状態とする)
- /unmei 側に3状態の分岐を実装:
  1. **未購入** → ティーザー+購入導線
  2. **購入済み・出生データ未入力** → 出生データ入力フォーム(指示書②のフォーム仕様)を表示。
     「あなたの設計図を描くために、生まれた日を教えてください」
  3. **購入済み・出生データあり** → 鑑定表示(生成中はローディング、ただし
     **タイムアウト60秒で「時間をおいて再度お試しください」表示**に切り替える。
     無限スピナー禁止)
- 出生データ保存時:natal_charts を計算 → 鑑定生成をトリガー → 完了後に表示

## タスク4:インタースティシャルの調整

**E2E実機確認で以下の逸脱を確認済み。優先的に修正すること:**

- **(4-1) 表示トリガーの修正**:現状、画面の最下部到達で発火している。
  仕様書①の通り「ページ全体の50%地点 かつ ¥499ペイウォールセクションより手前」
  かつ「スクロール停止300ms後」に修正。ペイウォール要素の offsetTop を取得し
  min(全体50%, ペイウォール手前) で判定する。¥499の購入判断に割り込まないことが
  この機能の存在意義なので、発火位置は最重要要件
- **(4-2) キャラクターが表示されていない**:ユニコーン・タカが画面に存在しない。
  原因を切り分けて修正:
  - 未実装の場合 → 仕様書①の通りプレースホルダ(単色シルエット+浮遊アニメ)を実装。
    アセット差し替え可能な構造(画像パスのコンフィグ化)にする
  - 実装済みだがレイアウト崩れで画面外の場合 → 配置修正
  - 合わせて、ぼかしホイール(回転アニメ)・星空背景・バッジ明滅など
    仕様書①「画面構成」の4層が揃っているか照合し、欠けを一覧報告してから実装
- デスクトップ(幅1024px以上)ではモーダル表示(最大幅420px・高さ85vh・角丸・
  暗色オーバーレイ)。モバイルレイアウトをそのまま収める。PCデザインを別に作らない
- 頻度制御(1ユーザー1回)の実機確認
- 旧タイプ名(動物ベース)の残存確認:リポジトリ全体を旧名で検索し、
  結果ページ・PDF・メール・シェアカード等に残っていれば称号マスタ参照に修正。
  検索結果の一覧を報告すること
