# 実装指示書②:購入後の出生データ取得フロー

## 背景

占いコンテンツ「運命の設計図」には出生データ(生年月日・出生時刻・出生地)が必要。
ただし**無料診断フローには絶対に追加しない**(完了率への影響リスクを避けるため)。
代わりに、¥499購入者に対して購入完了フローの中で任意入力してもらう。

取得したデータで裏側の出生図計算を先に済ませておき、後日のアップグレード訴求
「あなたの出生図が完成しました(+¥1,480)」に繋げる(指示書③参照)。

## 取得タイミングと導線

1. **¥499決済完了画面**:完了メッセージ+PDF送付案内の流れの中に、
   「レポートのお届けにあたって、よければ教えてください」として出生データ入力フォームを表示
2. **¥1,980直接購入時**:購入フロー内で同フォームを表示。
   こちらは鑑定生成に必須のため、生年月日のみ必須扱い(時刻・場所は引き続き任意)
3. **後から入力**:/me 内の設定またはプロフィール領域から入力・編集・削除できる導線を用意

## フォーム仕様

| 項目 | 必須/任意 | UI | 備考 |
|---|---|---|---|
| 生年月日 | ¥499後:任意 / ¥1,980:必須 | 年・月・日のセレクト(またはネイティブdate) | 未来日・120年以上前はバリデーション |
| 出生時刻 | 任意 | 時・分セレクト+「わからない」チェック | 「わからない」時は正午(12:00)仮定で計算し、`time_unknown` フラグを立てる |
| 出生地 | 任意 | 都道府県セレクト → 市区町村セレクト(2段階) | 市区町村→緯度経度のマスタ変換。未入力時は都道府県庁所在地、都道府県も未入力なら東京で仮計算し `place_unknown` フラグ |

- **スキップ可**を明示:「あとで入力」ボタンを常に表示。スキップしても¥499コンテンツ・PDF送付は
  一切制限しない(**購入物を人質に取らない**)
- フォーム下部に取得目的の明記(下記文言)

## プライバシー文言(必ずこの趣旨で表示)

> 出生データは、あなたのホロスコープ(出生図)の計算のみに使用します。
> それ以外の目的では利用しません。入力は任意で、設定からいつでも削除できます。

- **禁止事項**:「運営には見えません」等、実態と異なる表現は使わない
  (現状Supabaseに平文保存であり、閲覧不能ではないため)。
  将来カラム暗号化を実装した場合のみ文言をアップデートする
- プライバシーポリシーに「出生データ(生年月日・出生時刻・出生地)の取得と利用目的」の項を追記する
  (ポリシー本文の追記はテキスト差分を用意すること。公開反映は別途人間が確認)

## データモデル(Supabase)

新テーブル `birth_profiles`(users と 1:1):

```sql
create table birth_profiles (
  user_id uuid primary key references auth.users(id) on delete cascade,
  birth_date date not null,
  birth_time time,             -- null = 不明
  time_unknown boolean not null default false,
  prefecture text,
  city text,
  latitude double precision,   -- 市区町村マスタから解決した値
  longitude double precision,
  place_unknown boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

- **RLS必須**:本人のみ select / insert / update / delete 可。サービスロールは計算処理用に許可
- 削除UI(設定画面)からの削除時は、`birth_profiles` の行削除に加えて、
  生成済み鑑定キャッシュ(指示書③の `natal_readings` 等)も連動削除する
- 市区町村→緯度経度のマスタは静的JSON(国土地理院 or 総務省の公開データ由来)をリポジトリに同梱

## 入力後の裏側処理

- 生年月日が保存されたら、バックグラウンドで出生図の**計算のみ**実行して
  `natal_charts` (指示書③) に保存する(AI鑑定文の生成はまだしない=購入まで生成しない)
- 計算完了後、ユーザーに `natal_chart_ready` フラグを立てる。
  これが「あなたの出生図が完成しました」訴求(通知・/me内カード)のトリガーになる
- 通知の実装(LINE/メール)は本指示書のスコープ外。フラグとカード表示までを実装する
  - /me 内カード文言例:「⭐ あなたの出生図が完成しました — 続きを見る(+¥1,480)」
  - カードは¥499購入済み かつ `natal_chart_ready` かつ 未アップグレードのユーザーにのみ表示

## 計測イベント

- `birth_form_view` / `birth_form_submit` / `birth_form_skip`
- submit には `has_time` / `has_place` のプロパティを付与(入力率の把握)

## 受け入れ条件

- [ ] ¥499決済完了画面にフォームが表示され、スキップしても完了フローが正常終了する
- [ ] スキップ・未入力でも¥499コンテンツ/PDFに一切制限がかからない
- [ ] 時刻「わからない」で正午仮定+フラグ保存される
- [ ] 場所未入力時のフォールバック(県庁所在地→東京)が機能する
- [ ] RLSにより他ユーザーのデータが読めない(テストで確認)
- [ ] 設定画面から入力・編集・削除ができ、削除で鑑定キャッシュも消える
- [ ] 保存後に出生図計算が走り `natal_chart_ready` が立つ
- [ ] プライバシー文言が表示され、「見えない」等の虚偽表現が存在しない
